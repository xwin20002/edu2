## ✅ 已確認決策

- 本案是 **CC 新案／接管既有 repo**，不是重新建立 Git 專案。
- 不執行 `git init`，不重建 GitHub repo，不 force push，不覆寫既有 Git history。
- `edu3` 內容只作介面與 interaction reference，不可改名冒充小二上教材。
- 工作筆記放在 repo root，作為 `cc開案` 讀取與 `cc關案` 更新的 single source of truth。
- 外部 Obsidian dashboard 只存摘要與連結，不與 repo handoff 建立雙正本。
- 翰林是首套 baseline，但 canonical topic 與共用元件不得綁定出版社課次。
- 康軒／南一先保留切換位置；沒有正式來源時只顯示 planned，不建立推測內容。
- 多出版社建置 SOP 以 `WORKFLOW.md` 為準，並以 intake/mapping templates 與驗證腳本作 phase gate；edu4 後續沿用流程、不複製教材資料。
- Milestone A 是 Foundation 工作流程與資料入口，對應 `foundation.html`；Milestone B 是一般使用者看到的完整教學駕駛艙。A 完成不能代替 B，也不能因此宣告專案完成。
- 產製腳本應放入 `scripts/`，不可只留在 `/tmp`。
