標準 Pipeline

1. 開案與接管

先讀 repo handoff、Obsidian dashboard 與既有 SOP；不重新初始化 Git，不覆寫 history。

2. 資料 intake

核對出版社、學年度、課名、順序、來源與發布權限；未知內容保持 null。

3. NotebookLM

用指定帳號匯入已核對來源，產出簡報、資訊圖、字卡、測驗與影片，下載回 repo／本機。

4. YouTube

使用既有 CLI uploader，上傳為兒童內容與 unlisted，再將影片 ID 回填 manifest 與網站。

5. 批次產製

一個 golden unit 通過後，由 generator 產生全部內頁，不逐頁複製 HTML。

6. QA 與部署

執行 validator、桌機／手機 visual QA、互動測試、commit、push 與 production verification。

架構分層

穩定架構

  • 頁面資訊層級與學生／教師模式
  • 簡報、評量、影片與課堂工具
  • 國語直式、注音與發音 renderer
  • generator、CSS、JS、validator

可替換內容

  • title、focus、mission、objectives、flow
  • verticalSummary、vocabulary、characters
  • 出版社 mapping 與 source references
  • NotebookLM slides 與 YouTube ID

國語 Reference Rules

直式：以 edu1 L8 為準，固定使用 writing-mode: vertical-rl、text-orientation: mixed、行距 2.4、高度 180–320px，禁止加入 direction。
注音：不使用 <ruby> + vertical-rl；漢字與注音採手工 flex、排列使用 row-reverse，聲調放在最後一個注音符號右側。漢字 1.7em、注音 .62em、注音行距 1.25。
發音：使用 SpeechSynthesisUtterance，語言 zh-TW，速度 0.75。
內容：只放原創摘要或已取得發布授權的內容，不重製受版權保護的課文全文。
