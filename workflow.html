六階段標準 Pipeline

Phase 0 · Reuse／Scope／Continuity Preflight：開始六階段前，必須先盤點 reference GitHub、已安裝／共用 Skills、repo handoff 與 Obsidian 心得，產出 reuse / adapt / recollect / forbidden-content reuse map；同時標記 scopeClass 與 continuityStatus。未完成不得開始批次產頁。

Scope Class

unit：一課／一單元。
subject：單科多單元。
portfolio：跨科、半學期／整冊、多出版社或多 artifact pipelines。

Portfolio 必須每科獨立 Golden、artifact／QA ledger，並分 wave promotion。

Continuity Status

continuous：同一連續工作鏈。
resumed：跨 session 或中斷後接續。

Resumed 必須核對 Git、production、manifest、本機 assets、NotebookLM、YouTube、Obsidian、帳號與工具路徑，先退回上一個 verified gate。

1. 架構起始

從 edu1／edu3 GitHub repo 盤點並複製已驗證的工程架構：資訊層級、元件、CSS、JS、generator、validator 與部署方式。只搬架構，不搬其他年級教材內容。

Exit gate：空資料或 sample data 下，頁面結構與互動已達 reference 水準。

2. 重新蒐集資料

依目標年級、學期、出版社重新找正式或合法公開來源，核對課名、順序、學習目標、生字、注音與發布權限。

Exit gate：來源可追溯，未知內容維持 null，不從 edu1／edu3 推測教材。

3. 資料處理

整理 source pack，使用指定帳號建立 NotebookLM artifacts；影片下載後以 CLI uploader 上傳 YouTube，設定兒童內容與 unlisted。

Exit gate：簡報、圖卡、測驗、影片與 artifact manifest 均已拿回並記錄。

4. 資料換入架構

把新資料填入 content schema，由 renderer／generator 產生各課頁面；不逐頁複製 HTML，也不把教材內容寫死在共用元件。

Exit gate：全部課次可由資料重建，重新執行 generator 不會遺失手改內容。

5. 技術驗證

執行 validator、broken-link、asset、桌機／手機、學生／教師模式、發音、直式、注音、評量、影片與 production smoke test。

Exit gate：自動檢查與實際瀏覽器 QA 全部通過。

6. 品質確認與舊版對比

逐項對照 edu1／edu3 reference：資訊完整度、視覺層級、互動、教材可用性與課堂工具。新站必須達到一致水準，不能只因能開啟就宣告完成。

Exit gate：完成 quality parity checklist、差異紀錄與人工確認後才可關案。

NotebookLM／YouTube 執行踩坑與驗證

2026-07-13 用 edu1 小一數學 U4（既有合法資料、只缺影片收尾）完整跑過一次 Milestone B 產製鏈後半段（步驟 5–9），把踩過的坑固化成檢查點，115 三科正式跑 pipeline 時直接套用。

素材已下載 ≠ 已嵌入頁面

NotebookLM 素材下載到本機 assets/ 後，要實際打開單元 HTML 或用 grep／DOM 查詢確認素材路徑／YouTube embed src 真的出現在頁面，才能回報「已整合」；只確認檔案存在會漏掉「下載完但沒接進 HTML」。

git push 完成 ≠ 使用者立刻看得到

GitHub Pages／瀏覽器會快取舊版本，push 後直接開瀏覽器可能還顯示舊內容。驗證要用 curl 打 production URL，或 fetch(location.href,{cache:'no-store'})，或加 cache-busting query string，不能只看一次瀏覽器畫面下結論。

終端機中文亂碼 ≠ 資料損毀

youtube-upload 的 Python 腳本在 Windows PowerShell 下 print 中文會變亂碼（主控台編碼問題），但送進 API 的字串不受影響；要查 YouTube Studio 或 API 回傳值才準，不要單憑終端機顯示懷疑上傳錯誤。

交接筆記可能與實際檔案脫節

曾發生交接筆記寫「無 NotebookLM 資產」，但實際檔案早已產出並部署上線。接續任務前先用 grep／ls 核對實際檔案與已部署內容，發現落差要同步更正交接筆記。

影片狀態卡在 unknown

改到 NotebookLM 網頁手動下載，不要一直輪詢 API 等待狀態變化。

YouTube 每日配額（約 5–6 支）

批次上傳會中途卡住；未上傳單元先留明確占位文字（含預計恢復日期），配額重置後直接補傳，不必重新產生 NotebookLM 內容。

資料收集安全關卡

核心規則：「找到出版社」不等於「有課程內容」。每科必須依序通過 publisher-baseline-verified → official-outline-verified → unit-brief-verified → publication-ready → artifact-ready。關卡不足時，資料保持 null，不產頁、不產 NotebookLM、不上傳 YouTube。

可做什麼

  • 版本表：只設定 publisher／搜尋條件
  • 正式 outline：只登錄明列的主題與順序
  • 單元 brief：建立一個 subject Golden candidate
  • 發布／artifact ready：才開導覽或產媒體

不可跳過什麼

  • 舊學年、同名資源、詞彙 bank 不可補成新教材
  • 檔名／URL 寫 115 不算：必查 PDF／文件內文年度，不一致即 rejection
  • 公開 PDF／電子書不等於重製授權
  • 全冊 overview 不可當逐課影片
  • 國語 Golden 不可取代數學／生活 Golden
每次 promotion：必須同步 source registry、provenance log、工作筆記.md 與 validator；原始檔留在 ignored source/，Git 只提交必要 metadata。完整規格見 Content Source Gates。

架構分層

穩定架構

  • 頁面資訊層級與學生／教師模式
  • 簡報、評量、影片與課堂工具
  • 國語直式、注音與發音 renderer
  • generator、CSS、JS、validator

可替換內容

  • title、focus、mission、objectives、flow
  • verticalSummary、vocabulary、characters
  • 出版社 mapping 與 source references
  • NotebookLM slides 與 YouTube ID

Golden Samples

Golden Sample 是批次產製前的 quality gate，識別單位是「科目 × 年段 × 教材能力需求」。共用 shell 可以沿用，但不同科目與年段必須使用不同 presentation profile。115 資料尚未完成前，既有 Golden 只驗證 renderer／interaction，不驗證 115 課程內容。

Promotion rule：共用層變更需通過三科 Golden；科目 renderer 變更需通過該科 Golden。Golden 未通過，不得批次重建或宣告全部頁面已優化。

國語 Reference Rules

直式：以 edu1 L8 為準，固定使用 writing-mode: vertical-rl、text-orientation: mixed、行距 2.4、高度 180–320px，禁止加入 direction。
注音：不使用 <ruby> + vertical-rl；漢字與注音採手工 flex、排列使用 row-reverse，聲調放在最後一個注音符號右側。漢字 1.7em、注音 .62em、注音行距 1.25。
發音：使用 SpeechSynthesisUtterance，語言 zh-TW，速度 0.75。
內容：只放原創摘要或已取得發布授權的內容，不重製受版權保護的課文全文。
